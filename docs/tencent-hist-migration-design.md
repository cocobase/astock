# TencentDataSource 迁移至历史 K 线接口设计文档 (V2)

## 1. 设计动机
为了解决 `TencentDataSource` 无法获取历史数据、不支持复权以及在非交易时段数据获取不稳定的问题。计划将底层 API 从单一的“实时快照”切换为基于 JSON 的“历史 K 线”接口。

## 2. 接口矩阵与差异化策略

| 维度 | A股/港股历史接口 | 美股历史接口 | 实时快照 (Fallback) |
| :--- | :--- | :--- | :--- |
| **Endpoint** | `web.ifzq.gtimg.cn/appstock/app/fqkline/get` | `web.ifzq.gtimg.cn/appstock/app/usfqkline/get` | `qt.gtimg.cn/q=...` |
| **复权支持** | 不复权/前复权/后复权 | **仅前复权 (qfq)** | 不支持复权 |
| **数据格式** | JSON | JSON | 文本 (Tilde 分割) |
| **适用场景** | 历史补录、复权计算 | 历史补录 (仅前复权) | 当日实时、精确成交额 |

## 3. 详细设计方案

### 3.1 代码转换层 (Symbol Transformation)
程序需自动处理各市场代码的差异，尤其是美股后缀：
*   **A股**: `SH.600519` -> `sh600519` (转换前缀并转小写)
*   **港股**: `HK.00700` -> `hk00700`
*   **美股**: `US.AAPL` -> `usAAPL.OQ`
    *   **自动后缀逻辑**: 程序内部维护常用美股与其交易所后缀的映射表（如 `AAPL -> .OQ`, `NVDA -> .OQ`, `JPM -> .N`）。
    *   对于未知代码，默认优先尝试 `.OQ` (纳斯达克)，若请求失败则自动尝试 `.N` (纽交所) 或回退至快照。

### 3.2 字段映射与标准化
历史接口返回的 K 线数组需经过以下转换以符合系统标准：
*   **日期**: `data[0]` -> `YYYY-MM-DD` 字符串。
*   **OHLC**: 直接提取 `data[1-4]` 索引。
*   **成交量 (Volume)**:
    *   **A股**: `int(float(data[5]) * 100)` (腾讯接口 A 股原始单位为“手”，需转为“股”)。
    *   **港/美股**: `int(float(data[5]))` (原始单位即为“股”)。
*   **成交额 (Amount)**:
    *   由于历史接口缺失 `Amount` 字段，统一采用 **`Close * Volume`** 进行估算。
    *   若降级至快照接口，可直接提取原始 Amount。

**快照接口 (Snapshot) 通用索引结论**:
经过实测验证，以下索引在 A股、港股、美股的快照接口 (`qt.gtimg.cn`) 中具有高度一致性：
*   `fields[3]`: 当前价 (Close)
*   `fields[5]`: 今日开盘价 (Open)
*   `fields[33]`: 今日最高价 (High)
*   `fields[34]`: 今日最低价 (Low)
*   `fields[6]`: 成交量 (Volume)
*   `fields[37]`: 成交额 (Amount)
> *注：A 股成交量单位为手，成交额单位为万元；港美股成交量单位为股，成交额单位为原始货币。*

### 3.3 混合抓取逻辑 (Hybrid Logic)
1.  **Step 1: 历史优先** - 根据请求日期构造参数，调用对应市场的 `fqkline` 或 `usfqkline`。
2.  **Step 2: 响应校验** - 检查 JSON 返回码 `code == 0` 及数据是否存在。
3.  **Step 4: 降级/补充** - 
    *   若历史接口无数据（如美股接口延迟或代码错误），则调用 `qt.gtimg.cn` 尝试获取。
    *   若需要获取“今日”的完整财务字段（如市盈率、换手率），需并行调用快照接口补充字段。

## 4. 迁移步骤
1.  **辅助层实现**: 编写 `_build_hist_url` 和 `_transform_symbol` 辅助函数。
2.  **核心重构**: 修改 `fetch_daily_kline` 主逻辑，集成 JSON 解析器。
3.  **单位转换验证**: 通过 `tests/debug_tencent_hist.py` 验证 A 股成交量计算的准确性。
4.  **美股联调**: 测试常用美股标的的后缀自动匹配能力。

## 5. 预期收益与风险
*   **收益**:
    *   支持 `adj='qfq'`，使腾讯数据源可以无缝替代 `yfinance` 进行历史补录。
    *   系统稳定性大幅提升，不再依赖“当前必须在交易时间内”这一约束。
*   **风险**:
    *   美股交易所后缀变动。*对策：建立后缀缓存或自动重试逻辑。*
    *   成交额估算误差。*对策：在文档中明确该字段为计算推导值。*
