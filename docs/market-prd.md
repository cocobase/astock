# 多市场收盘日K线数据稳定获取系统需求文档

## 1. 文档概述

### 1.1 文档目的

明确A股、港股、美股指定股票**收盘固化日K线数据**的自动化、高可靠获取系统的需求边界、功能规则与交付标准，实现多数据源主备自动切换、全链路可追溯、数据统一CSV持久化，结合交易日历精准控制数据下载时机，满足个人量化回测、复盘研究的稳定数据需求。

### 1.2 适用范围

- 覆盖市场：A股、港股、美股

- 数据范围：交易日收盘后固化的完整日K线数据（含复权价格），非交易日不执行下载操作

- 适用用户：个人量化投资者、研究人员

- 运行环境：Windows/macOS/Linux，Python 3.8+

## 2. 核心业务目标

1. 高可用：通过多数据源主备架构，结合交易日历精准触发，保障正常开盘标的的日K数据获取成功率≥99%

2. 自动化：支持每日定时执行，先校验当日是否为对应市场交易日，非交易日不执行下载；失败自动重试、自动切换备用数据源，无需人工干预

3. 可追溯：完整记录每条数据的下载链路、数据源、执行结果，同时记录交易日校验结果及下载触发状态，支持问题排查

4. 标准化：数据统一字段、统一CSV格式存储，兼容主流量化回测框架；交易日历判断逻辑统一，确保各市场下载时机精准

## 3. 核心功能需求

### 3.1 数据源管理模块

1. **多数据源优先级配置**：支持按「主数据源→备用数据源1→备用数据源2」的固定优先级调用，可通过配置文件调整顺序；新增交易日历数据源（exchange_calendars），作为独立校验数据源，优先级高于所有数据获取数据源（仅用于交易日判断）
        

    - 主数据源（全市场优先）：富途OpenAPI（合规稳定、全市场覆盖、数据质量最高）

    - 备用数据源1（分市场适配）：A股=Tushare Pro，港股/美股=yfinance

    - 备用数据源2（全市场兜底）：AKShare

    - 交易日历数据源：exchange_calendars（https://github.com/gerrymanoim/exchange_calendars），用于校验A股、港股、美股当日是否为交易日，支撑下载触发逻辑

2. **代码格式适配**：自动将统一格式的股票代码转换为对应数据源支持的编码规则；同时自动匹配exchange_calendars支持的市场编码（如A股对应SSE/SZSE、港股对应HKEX、美股对应NYSE/NASDAQ），无需人工调整
      

3. **数据源健康检查**：程序启动时自动校验各数据源的连通性与权限有效性，包括exchange_calendars库的可用性及市场编码匹配度，提前预警不可用数据源
      

### 3.2 核心数据获取模块

1. **数据范围与触发条件**：仅在对应市场的交易日，获取收盘后的**固化完整日K数据**，不处理盘中实时数据、非交易日不执行任何下载操作；支持历史数据补录（补录时需先通过交易日历校验目标日期是否为交易日）与每日增量更新
     

2. **交易日校验逻辑**：通过exchange_calendars库，按市场分别校验当日（或补录日期）是否为交易日，规则如下：
        

    - A股：调用exchange_calendars中SSE（上交所）、SZSE（深交所）日历，任一市场开盘即判定为A股交易日（针对全市场标的统一触发）

    - 港股：调用exchange_calendars中HKEX（港交所）日历，判定当日是否为港股交易日

    - 美股：调用exchange_calendars中NYSE（纽交所）或NASDAQ（纳斯达克）日历，任一市场开盘即判定为美股交易日（针对全市场标的统一触发）

    - 若exchange_calendars调用失败，默认按历史交易日规律（排除周末、法定节假日）兜底判断，同时记录告警日志

3. **复权规则**：默认获取前复权（qfq）数据，支持通过配置切换为后复权（hfq）、不复权（none）
      

4. **统一输出字段**：所有数据源返回的数据统一转换为以下标准字段，无缺失字段用`null`填充
      

### 3.3 主备自动切换与重试机制

1. **执行前置校验**：每日定时执行时，先通过exchange_calendars校验对应市场当日是否为交易日；非交易日则记录日志（无需执行下载），直接终止当日该市场的数据获取流程；交易日则进入单标的执行链路。
      

2. **单标的执行链路**：针对单只股票+单个交易日，按优先级顺序调用数据源，规则如下：


    1. 调用当前优先级数据源，先执行股票代码格式适配

    2. 调用失败（超时、接口报错、返回空数据、核心字段缺失）时，自动重试2次，重试间隔1秒

    3. 重试仍失败，自动切换至下一优先级数据源

    4. 任意数据源调用成功，立即终止链路，进入数据存储流程

    5. 所有数据源全部失败，记录错误信息，不中断其他股票的执行流程

3. **超时控制**：单个数据源接口调用超时时间默认10秒，可通过配置调整；exchange_calendars调用超时时间单独配置，默认5秒，超时则触发兜底交易日判断逻辑
      

4. **异常隔离**：单只股票、单个数据源的异常，不影响整体程序运行与其他标的的数据获取；exchange_calendars调用异常不影响数据获取数据源的正常执行（仅触发兜底判断）
