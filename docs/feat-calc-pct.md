# 需求与设计文档：计算股票收盘涨幅 (V2 - 本地数据优先)

## 1. 需求说明 (Requirements)

### 1.1 功能背景
系统已定期将 A 股、港股、美股的历史 K 线数据同步至本地 `data/` 目录。为了分析当前持仓或关注列表的最新表现，需要从本地存储中提取最后两个交易日的数据，计算最新涨幅并汇总。

### 1.2 核心功能
1.  **数据来源**：完全依赖本地 `data/` 文件夹（支持 A-Share, HK, US 子目录）中的历史数据文件。
2.  **股票范围**：从 `config/config.yaml` 中读取配置的股票列表。
3.  **计算逻辑**：
    *   读取每个股票数据文件的**最后两行**。
    *   `pct-change` (涨幅 %) = `(last_close - second_last_close) / second_last_close * 100`。
    *   字段提取：`open`, `high`, `low`, `close` 均取最后一行。
4.  **输出规范**：
    *   文件路径：`data/pct-change.csv`。
    *   字段列表：`symbol`, `open`, `high`, `low`, `close`, `pct-change`。
    *   精度控制：所有数值保留 **2 位小数**。
5.  **交互方式**：通过主程序参数 `python -m src.main --calc-pct` 触发。

---

## 2. 系统设计 (Design)

### 2.1 架构设计
该功能作为“离线数据处理模块”实现，集成在 CLI 流程中。

*   **配置解析**：复用 `ConfigLoader` 获取股票列表及市场分类。
*   **存储访问**：在 `src/core/storage.py` 中寻找或增加读取本地文件末尾的方法。
*   **计算引擎**：遍历股票，定位 `data/{Market}/{Symbol}.csv`，读取末尾两行进行计算。
*   **输出汇总**：使用 Pandas 构造结果集并导出 CSV。

### 2.2 核心处理流程
1.  **Load Config**: 获取 `stocks` 配置。
2.  **File Mapping**: 根据市场类型映射到具体的物理路径。
3.  **Data Extraction**: 高效读取文件末尾（避免加载全量大数据文件）。
4.  **Calculation**: 执行涨幅算法，处理停牌或数据不足的边界情况。
5.  **Export**: 格式化数值并写入 `data/pct-change.csv`。

## 3. 测试计划 (Test Plan)

### 3.1 测试场景
| 场景 ID | 测试点 | 预期结果 |
| :--- | :--- | :--- |
| TC-01 | **正常路径** | 成功读取本地 A 股/港股文件，计算出正确的涨幅并生成 CSV。 |
| TC-02 | **文件缺失** | 某股票在 `data/` 下无文件，程序记录日志并跳过，不报错。 |
| TC-03 | **数据不足** | 股票只有 1 天数据（如刚上市），涨幅设为 0.00。 |
| TC-04 | **数值精度** | 检查 CSV 结果，所有数值均为 2 位小数（如 `15.00`）。 |
| TC-05 | **多市场支持** | 验证 A-Share, HK, US 三个子目录下的文件都能被正确识别。 |

### 3.2 验证方法
*   运行 `python -m src.main --calc-pct`。
*   比对 `data/pct-change.csv` 中的数据与本地原始 CSV 末尾数值的准确性。
