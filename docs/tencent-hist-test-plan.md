# 腾讯历史 K 线接口 (fqkline) 测试方案

## 1. 测试背景
当前 `TencentDataSource` 使用的实时快照接口 (`qt.gtimg.cn`) 仅能获取最新交易日数据，无法满足非交易时段补录或追溯历史的需求。本方案旨在验证腾讯历史 K 线接口 (`web.ifzq.gtimg.cn`) 的可用性，为数据源重构提供依据。

## 2. 待测接口技术规范
*   **接口地址**: `https://web.ifzq.gtimg.cn/appstock/app/fqkline/get`
*   **请求方式**: GET
*   **核心参数**: `param={symbol},day,{start_date},{end_date},{max_len},{adj_type}`
    *   `symbol`: 市场前缀 + 代码 (如 `sh600519`, `hk00700`, `usAAPL`)
    *   `start_date`: 开始日期 (`YYYY-MM-DD`)
    *   `end_date`: 结束日期 (`YYYY-MM-DD`)
    *   `max_len`: 获取条数 (如 `320`)
    *   `adj_type`: 复权类型 (`qfq` 为前复权, `hfq` 为后复权, 不传为不复权)

## 3. 测试矩阵与验收标准

| 测试项 | 场景说明 | 验收标准 |
| :--- | :--- | :--- |
| **全市场覆盖** | 分别请求 A股/港股/美股标的 | 返回格式一致的 JSON 数据，包含 `code: 0` |
| **历史追溯性** | 请求 3 天前的历史交易日数据 | 返回结果包含指定日期的 K 线数组 |
| **复权一致性** | 对比 `qfq` 与不复权的价格差异 | 在已知除权标的上，`qfq` 价格应与 `yfinance` 趋势一致 |
| **字段完整性** | 检查返回的 List 元素 | 必须包含: 日期, 开, 收, 最高, 最低, 成交量, 成交额 |
| **数据单位校准** | 验证成交量和成交额的原始单位 | 确定 A 股是否需 `*100` (手转股)，成交额是否为万元 |
| **频率限制测试** | 模拟 1 秒内连续 10 次请求 | 接口应保持稳定，不出现 403 或验证码拦截 |

## 4. 测试实施步骤
1.  **脚本开发**: 编写 `tests/debug_tencent_hist.py` 调用接口并打印原始数据结构。
2.  **数据采集**: 采集贵州茅台 (A)、腾讯控股 (HK)、苹果 (US) 的 2026-04-03 历史数据。
3.  **横向对比**: 将采集到的历史数据与已有的 CSV 固化数据进行 OHLC 值比对。
4.  **结果评估**: 确认字段索引位置（如：索引 0 是日期，索引 1 是开盘等）。

## 6. 测试结果总结 (2026-04-06)

| 市场 | 接口可用性 | 字段索引 (OHLCV) | 备注 |
| :--- | :--- | :--- | :--- |
| **A股** | 优 (支持 qfq) | 0:日期, 1:开, 2:收, 3:高, 4:低, 5:量 | 成交量单位为“手”，成交额缺失 |
| **港股** | 优 (支持 qfq) | 0:日期, 1:开, 2:收, 3:高, 4:低, 5:量 | 索引 6 为回购/公告字典，成交额缺失 |
| **美股** | 差 (不可用) | N/A | `fqkline` 接口返回空数据，需保留快照接口 |

### 关键发现
1. **成交额缺失**: 历史接口不返回成交额（Amount），需通过 `Close * Volume` 估算，或在需要精确成交额时降级到快照接口。
2. **单位差异**: A股成交量为“手”（需 * 100 转为“股”以匹配系统标准），港股成交量为“股”。
3. **复权开关**: 传入 `qfq` 时，返回的 Key 依然可能是 `day`，需做兼容性 Key 检查。
4. **美股处理**: 确认腾讯历史 API 不支持美股，US 市场必须继续使用 `qt.gtimg.cn` 实时快照（仅支持当日）。
